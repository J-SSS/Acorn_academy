배열의 선언과 내부 반복문과 정렬

타입이 명확하지 않아서 발생하는 문제 해결을 위한 const 선언

  • (자바) ArrayList list = new ArrayList(); list=13;(타입이 명확해서 이렇게 치면 컴파일오류)
  • (JS) let list = new Array(); list=13; (모든 타입이 가능하기에 참조가능)
  • list=13인걸 까먹고 list Array 자료형이라 생각하다가.. list[3]을 입력하면 undefined가 떠버림
  • =>> 자료형을 선언할 때 상수로 선언해서 해결하도록함.. 강제는 아니고 그냥 응용임

배열의 내부 반복문(iteration 함수)

콜백함수 선언시 iterator 객체를 이용해 배열의 모든 필드에서 콜백 함수를 실행하는 반복문

내부 반복문은 기본 함수를 정의하기 때문에 재사용성이 높고 가독성이 좋다.(자바에서는 선호하지않음)

  • forEach((e,index,array)=>{}) : 현재 배열을 검사하거나 소비하는 반복문
  • map((e,index,array)=>{return modify e}) : 각 요소를 변경한 수정된 새로운 배열을 반환
  • reduce((total, e, i, array)=> total 연산 e, total=val ); return total : 모든 요소를 연산한 하나의 결과를 반환
  • filter((e,i,array)=>(e 비교연산 )); 조건에 맞는 요소만 남긴 새로운 배열을 반환
  • every((e,i,array)=>(e 비교연산)); return boolean
  • some((e,i,array)=>(e 비교연산)); return boolean
  • find((e,i,array)=>(e 비교연산)); return 찾은요소
  • findIndex((e,i,array)=>(e 비교연산)); return 찾은 요소의 index
  • includes(찾는 값); return boolean
  • indexOf(찾는 값, start index=0); return index
  • LastindexOf(찾는 값, start index=0); return index

배열의 정렬 함수

배열에 있는 요소가 정렬이 될 때 까지 반복하는 함수.. 기본은 문자열을 정렬

  • sort() : 문자열 오름차순 정렬
  • reverse() : 문자열 내림차순 정렬
  • sort((a,b)=>a-b) : 수의 오름차순 정렬
  • sort((a,b)=>b-a) : 수의 오름차순 정렬
  • Object.sort((a,b) => a.필드 - b.필드) : 자료형의 필드를 오름차순 정렬
  • Object.sort((a,b) => b.필드 - a.필드) : 자료형의 필드를 내림차순 정렬
